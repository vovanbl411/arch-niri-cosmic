# Руководство по внесению вклада

Спасибо, что рассматриваете возможность внести вклад в проект PKGS-ARCH! Любые улучшения и дополнения приветствуются.

## Структура проекта

Проект организован по категориям пакетов:

- `core/` - базовые системные пакеты
- `desktop/` - пакеты рабочего стола и приложений
  - `niri.txt` - пакеты для среды рабочего стола Niri
- `development/` - утилиты для разработки
- `fonts-themes/` - шрифты и темы оформления
- `hardware/` - драйверы и прошивки
- `virtualization/` - пакеты виртуализации
- `aur/` - пакеты из AUR
  - `aur.txt` - основные пакеты из AUR
  - `cosmic.txt` - пакеты для среды рабочего стола COSMIC (в AUR)


## Как внести вклад

### Добавление новых пакетов

1. Определите подходящую категорию для пакета
2. Откройте соответствующий файл (например, `desktop/apps.txt`)
3. Добавьте имя пакета в новую строку
4. Убедитесь, что пакет не дублирует уже существующий

### Создание новой категории

Если вы хотите добавить новую категорию пакетов:

1. Создайте новый подкаталог в корне проекта
2. Добавьте файл `.txt` с перечнем пакетов
3. Обновите документацию (README.md) с описанием новой категории

### Проверка пакетов

Перед добавлением убедитесь, что:

- Пакет доступен в соответствующем репозитории (main, extra, community, AUR)
- Имя пакета указано правильно
- Пакет не вызывает конфликта с уже установленными

## Стиль оформления

- Используйте по одному имени пакета на строку
- Пакеты в файле можно группировать по функциональности
- Добавляйте комментарии, если необходимо объяснить выбор пакета

## Процесс отправки изменений

1. Сделайте форк репозитория
2. Создайте новую ветку: `git checkout -b feature/описание-изменений`
3. Внесите изменения
4. Зафиксируйте изменения: `git commit -m 'Добавил пакеты в категорию desktop'`
5. Отправьте изменения: `git push origin feature/описание-изменений`
6. Создайте Pull Request

## Тестирование

После добавления новых пакетов рекомендуется проверить:

- Корректность имени пакета
- Возможность установки пакета
- Отсутствие конфликтов с другими пакетами

## Сообщество

Если у вас есть вопросы или предложения, не стесняйтесь создавать issue для обсуждения.
